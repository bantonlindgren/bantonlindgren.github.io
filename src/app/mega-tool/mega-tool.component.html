Mega Tool

​Me and my fellow technical artists wrote a Maya tool for our game projects group that would be referred to as the Mega Tool. The tool is meant to streamline a big part of the art creation pipeline.

A brief outline of the features that I have been involved in writing:

Apply material

Lets the user choose a folder with targa files and automatically creates a material for the model and matches the correct color channels of our textures to the correct channels of the material based on our name and texture packing standards.

Export FBX

With one button click the user can export a triangulated version of the mesh as an fbx along with the targa files into a location so they can easily be used in unreal by our level designers.

The tool simultaneously exports a copy of the fbx to our game engine along with the textures that are being converted to dds images in this stage (the DDS converter has a page of it's own).

When the user tries to export the mesh the files will first be validated and if any problems are detected the tool will say which files appear to be wrong and in what way (more on the file validation below).

Show model in showroom

This button will open the model in our game engine so one can see how our shaders and lighting will interact with the mesh and the material.

Ducky Project Validator

​This tool is a file validator that I wrote in C#. It looks through and validates all files in a given folder and its subfolders.

We validate the files to make sure they adhere to our standards. This includes:

Checking that we have no files with other extentions than .tga, .dds or .fbx.

Checking that all files adhere to our name standard. We have a rather detailed name standard so this check is not entirely trivial. I wrote the checks using regular expressions and wrote a hefty amount of unit tests to make sure it was working properly.

Check that our textures are power of two sizes, not larger than 4k and if the entire image is only one color - not larger than 16x16 px.

DDS Converter

This is a very minimal tool that I wrote that just takes a to and a from path. With this information it simply takes all the targa files in the from folder and it's subfolder, convertes them to DDS and puts them in the to folder.

If the to folder does not exist since before, it is created. A neat little feature in this tool is that it will check if the source image is only a single color and if so it will resize the output image to 16x16 pixels.

The motivation for writing this tool is that our custom game engine uses dds files. The tool is specifically constructed to be easy to integrate with our Mega Tool in maya.
